補足：Omega II

Omega II は -2、-1、0、+1、+2 の 5 種類のカード値を使う平衡マルチレベルカウントシステムです。A はメインカウントでは 0 とし、ace side count で別途追跡することで、ベットとインシュランスの判断をより精確にします。構造は Hi-Opt I に似ていますが、カード値の割り当てが異なります。

Omega II の設計目標は低カード（特に 4–6）に最も強いシグナル（+2）を与えつつ、高カードをより細かく区別することです。10/J/Q/K は -2、9 は -1、A は 0 として別途追跡します。これによりメインカウントが 10 点カードの密度をより正確に反映し、A の情報を side count で補完してベットとインシュランス判断を完結させます。

Ace Side Count と Ace Adjustment

Hi-Opt I と同様に、Omega II のメインカウント（RC）では A を 0 とし、ace side count で見た A を別途追跡します。Ace side count によってアプリは ace deficit（実際に出た A が理論予想より少ない枚数）を計算し、以下の式でベットとインシュランス用の true count を修正します：

指標意味計算式 / 補足
Running CountAce-neutral メインカウントA = 0
True Countプレイ判断用RC / 残りデッキ数
Ace Seen見た A の枚数Side count
Ace Deficit理論より少ない A の枚数usedCards / 13 - aceSeen
Adjusted RCace deficit と k で修正した RCRC + k × Ace Deficit
Betting TCベット判断用Adjusted RC / 残りデッキ数
Insurance TCインシュランス判断用Betting TC と同じ

式中の k はアプリ設定の ace adjustment パラメータで、デフォルトは 2 です。k を上げると ace deficit が Betting TC に与える影響が大きくなり、下げると小さくなります。

アプリでは ace side count のオン/オフを切り替えられます。オフ時はメインカウントの True Count をすべての判断に使用し、オン時はベットとインシュランスを修正済み Betting TC に切り替えます。このスイッチは Hi-Opt I と同じ設計で、Game Settings で調整できます。

Omega II のプレイ判断にはメインカウントから換算した True Count を使用し、ace adjustment の Betting TC は使いません。Ace Side Count はベットとインシュランスの 2 つの判断のみに影響します。ace side count を有効にした場合の k のデフォルトは 2 で、Game Settings で調整できます。
