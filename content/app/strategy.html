策略

策略功能讓使用者針對每個算牌系統建立自訂的 deviation 規則集，而不受限於預設的 index spec。你可以從系統預設出發，複製一份後修改個別規則，測試不同 index 門檻對決策的影響。

策略頁面與系統選擇

進入策略頁面後，頂部可切換算牌系統：Basic、Hi-Lo、KO、Red 7、FELT、Zen、Hi-Opt I、Omega II，以及 H17 / S17 篩選器。每個系統在不同莊家規則下各有一份預設策略，顯示為上鎖的「Preset」卡片。

預設策略對應 App 遊戲與訓練模式預載的 index spec，可以直接點進去查看完整的 deviation 規則表，但無法編輯。若想修改規則，需要先建立一份自訂策略。

建立自訂策略

點右下角的「新增策略」，輸入名稱後，App 會以目前選取系統和 H17/S17 設定的預設策略為基礎複製一份新的可編輯策略。建立後可以直接進入編輯，也可以之後再修改。

自訂策略卡片右側有刪除按鈕，可移除不再使用的策略，預設策略無法刪除。

查看與編輯策略規則

點進一份策略後，會看到按 Hard / Soft / Pair 分頁的策略表格，表格欄對應莊家明牌（2–A），列對應玩家手牌點數。每個格子顯示目前的動作（Hit、Stand、Double、Split、Surrender）。對於自訂策略，偏離預設基本策略的格子會以顏色標記，方便一眼看出哪些情境被修改過。

點格子可以開啟規則編輯介面，設定以下項目：

  • 手牌類型：Hard、Soft、Pair、Insurance
  • 莊家明牌：2–10 或 A
  • 玩家手牌點數（Insurance 不需填）
  • TC / RC 門檻：觸發此規則的 count 條件
  • 比較方向：>= 或 <
  • 覆蓋動作：符合條件時執行的動作
  • 備註（選填）：記錄這條規則的來源或理由
KO 系統的門檻以 Running Count（RC）為單位，其他系統（Hi-Lo、FELT、Zen、Hi-Opt I、Omega II 等）以 True Count（TC）為單位。編輯介面會依目前策略的系統自動顯示對應的欄位標籤。

套用自訂策略到遊戲

建立自訂策略後，到 Settings 的「Advisor mode」區塊，在下拉選單中選取對應的策略名稱，遊戲與 Decision Training 就會依照自訂規則給出建議。如果選「Default」，則使用系統內建的預設 index spec。

使用情境

測試不同 Index

把某條 deviation 的門檻調高或調低，觀察在 Game 模式和 Decision Training 中觸發頻率的變化。

簡化版 Spec

從預設策略移除幾條低頻或難記的 deviation，建立一套適合自己目前記憶容量的簡化版，再逐步恢復。

對照不同資料來源

如果你在其他書籍或工具中看到不同的 index 值，可以複製預設後逐條對照修改，用 App 驗證差異。
