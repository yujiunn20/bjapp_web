補足：Zen Count

Zen Count は -2、-1、+1、+2 の 4 種類のカード値を使う平衡マルチレベルカウントシステムです。Hi-Lo と比べて A を独立して -1 でカウントするため、A の消費が RC に直接反映され、最も有利な低カード（4–6）には強い +2 シグナルを与えます。

Hi-Lo は A と 10 を同じ -1 として扱いますが、A の blackjack への貢献は通常の 10 よりはるかに大きいです。Zen は A を -1、10/J/Q/K を -2 と分けることで、残りデッキの A の枚数を RC から部分的に読み取れるようにし、Hi-Opt I のような別個の ace side count なしに精度と複雑さのバランスを取っています。

True Count 換算

Zen Count は平衡システム（1 デッキ分で RC が 0 に戻る）のため、Hi-Lo と同じ方法で True Count 換算を行います：TC = RC / 残りデッキ数。カウント幅が -2 から +2 のため RC の変動が Hi-Lo より大きく、TC 値も比例して大きくなります。

Zen Count は Hi-Lo より 16 vs 9 と 16 vs A のサレンダー deviation が多くなっています。Zen の A が -2 ではなく -1 のため、残りデッキに A が多めの場合に TC が相対的に高くなり、危険なディーラーの組み合わせをより早く識別できます。Hi-Lo の deviation index を Zen Count に直接適用することはできません。
