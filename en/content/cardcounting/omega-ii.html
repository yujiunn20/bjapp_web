Extra: Omega II

Omega II is a balanced multi-level card counting system using five values: -2, -1, 0, +1, and +2. Aces count as 0 in the main count and are tracked separately with an ace side count for more precise betting and insurance decisions — a structure similar to Hi-Opt I but with different card value assignments.

Omega II is designed to give the strongest signal (+2) to low cards — especially 4–6 — while providing finer high-card distinctions: 10/J/Q/K count as -2, 9 counts as -1, and A is zeroed out and tracked separately. This lets the main count more accurately reflect the density of 10-value cards, while ace information is supplemented via side count for complete betting and insurance decisions.

Ace Side Count and Ace Adjustment

Similar to Hi-Opt I, Omega II's main count (RC) treats A as 0 and separately tracks aces seen with an ace side count. The side count lets the app calculate the ace deficit — how many fewer aces have appeared than expected — and uses the following formula to adjust the true count for betting and insurance:

MetricMeaningFormula / Note
Running CountAce-neutral main countA = 0
True CountUsed for playing decisionsRC / decksRemaining
Ace SeenNumber of aces seenSide count

Insurance TCUsed for insurance decisionsSame as Betting TC

The k in the formula is the ace adjustment parameter in app settings, with a default value of 2. Increasing k amplifies the ace deficit's effect on Betting TC; decreasing it reduces the effect.

The app allows toggling ace side count on or off. When off, the main True Count is used for all decisions. When on, betting and insurance switch to the adjusted Betting TC. This toggle works the same as Hi-Opt I and can be adjusted in Game Settings.
