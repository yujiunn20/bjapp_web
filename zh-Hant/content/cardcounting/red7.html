補充：Red 7

Red 7 是一套不平衡算牌系統，設計上介於 Hi-Lo 和 KO 之間。它把紅色的 7（♥ 和 ♦）計為 +1、黑色的 7（♠ 和 ♣）計為 0，以這個「半個 7」的不平衡製造出省略 True Count 換算的效果。

一般的平衡系統（如 Hi-Lo）需要把 Running Count 除以剩餘牌組數才能得到 True Count。Red 7 透過只計算紅色 7，讓每副牌多出 +2 的淨計數（因為每副 4 張 7 中，只有紅色 2 張算 +1），造成適度不平衡，使得 RC 本身就能反映牌堆深度的資訊，可直接用於判斷。

和 Hi-Lo 相比，Red 7 的差異只在 7 的處理：Hi-Lo 的 7 全部是 0，Red 7 把其中的紅色 7 計為 +1。這讓 Red 7 只需要額外辨識花色，就能在 Hi-Lo 的基礎上使用不同的判斷門檻。

IRC（Initial Running Count）

由於 Red 7 不平衡，每副牌多出 +2 的計數，6 副牌時總體多出 +12。為了讓 RC 門檻在不同牌組數下具備一致的意義，通常會設 IRC = -2 × 牌組數，6 副牌時 IRC = -12，讓牌堆接近一半（正常優勢反轉點）時 RC 落在 0 附近。App 的 Red 7 訓練模式支援設定起始 RC。

Red 7 在省略 True Count 換算的同時，比 KO 更接近 Hi-Lo 的判斷精度，因為它的不平衡量較小（每副 +2 vs KO 每副 +4）。代價是需要識別 7 的花色，增加了一點追蹤負擔。
